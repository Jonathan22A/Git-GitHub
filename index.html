Comandos para la configuración de Git

git --version

Muestra si está Git instalado correctamente, mostrando la versión del mismo.

git config --global user.name "UsuarioDeGitHub"

El config --global Sirve para que la configuración que agregamos a continuación se efectúe de manera global
y no por proyecto. Con (user.name) agregamos el Usuario a Git (tiene que ser el mismo usuario que colocamos en GitHub).

git config --global user.email "CorreoDeGitHub"

El config --global Sirve para que la configuración que agregamos a continuación se efectúe de manera global
y no por proyecto. Con (user.email) agregamos el email a Git (tiene que ser el mismo usuario que colocamos en GitHub).

git config --global core.editor "code --wait"

El config --global Sirve para que la configuración que agregamos a continuación se efectúe de manera global
y no por proyecto. Y (core.editor) para indicarle a Git que VSCode es nuestro editor de texto por defecto.
(code --wait) es para que la terminal se quede esperando hasta que nosotros cerremos nuestro editor de texto.

git config --global core.autocrlf “true”

El config --global Sirve para que la configuración que agregamos a continuación se efectúe de manera global
y no por proyecto. Y (core.autocrlf) para que Git sepa como tratar los saltos de línea. Si utilizas una mac
o tienes linux se reemplaza "true" por "input".

git config --global -e

El config --global Sirve para que la configuración que agregamos a continuación se efectúe de manera global
y no por proyecto. Y (-e) Ver el archivo de configuración Global.

git config --global -h

El config --global Sirve para que la configuración que agregamos a continuación se efectúe de manera global
y no por proyecto. Y (-h) para ver todas las configuraciones que pueden realizarse en Git.

Comandos básicos

ls

Permite poder listar todos los archivos y carpetas que se encuentren dentro de una determinada carpeta,
excluyendo las carpetas y archivos ocultos.

ls -a

Permite poder listar todos los archivos y carpetas que se encuentren dentro de una determinada carpeta,
incluso las carpetas y archivos ocultos.

pwd

Para saber en qué carpeta nos encontramos actualmente.

cd

Para poder movernos entre carpetas. Precisamente para poder ingresar a las carpetas.

cd ..

Para poder movernos entre carpetas. Precisamente para poder salir de las carpetas.

mkdir "NombreNuevaCarpeta"

Para poder crear una nueva carpeta desde la terminal.

code .

Para abrir el editor de texto desde la terminal.

cat "NombreDelArchivo"

Muestra lo que contiene el archivo.

Comandos básicos de Git

git init

Comando para inicializar un Repositorio.

git status

Nos muestra el estado actual de nuestro repositorio.

git add "NombreDelArchivo"

Añade un cambio del directorio de trabajo en el entorno de ensayo. De este modo, indica a Git que
quieres incluir actualizaciones en un archivo concreto en la próxima confirmación.

git commit -m "NombreSignificativo"

Captura una instantánea de los cambios preparados en ese momento del proyecto. Las instantáneas confirmadas
pueden considerarse como versiones seguras de un proyecto.

git rm "NombreDelArchivo"

Para eliminar un archivo.

git mv "NombreAteriorDelArchivo" "NombreNuevoDelArchivo"

Par cambiar el nombre de un archivo.

git diff

Muestra lo que que se modificó en los archivos.

git log

Muestra el historial de los commit junto con los datos de la persona que los realizó.

git log --oneline

Muestra el historial de los commit en una línea.

Comandos de ramas en Git (Branches)

git branch

Para saber en qué rama nos encontramos actualmente.

git checkout -b "NombreDeRama"

Para crear una nueva rama.

git checkout "NombreDeRama"

Para cambiar de rama.

git merge "NombreDeRama"

Para traer todos los cambios realizados de una rama a la otra. (Importante: El comando debe ser escrito
en la rama donde queremos copiar los datos de la otra rama.)

Comandos para vincular Git con GitHub

git remote add origin "LinkDelRepositorioDeGitHub"

Con (git remote) podemos indicar si vamos a tener un servidor remoto en el cual subir los cambios.
Y con (add origin) le estamos indicando de donde nosotros tenemos que obtener el código y también a donde
tenemos que subir los cambios que realicemos en el mismo.

git push -u origin "NombreRama"

Para poder crear una rama en GitHub donde ir guardando los cambios.

git push

Para subir los cambios respecto con la rama que estemos trabajando.
